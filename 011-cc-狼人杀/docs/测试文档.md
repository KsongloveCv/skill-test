# 狼人杀网页游戏 — 测试文档

## 1. 测试范围

| 类别 | 覆盖范围 |
|------|---------|
| 功能测试 | 游戏完整流程、角色技能、胜负判定、AI决策 |
| UI 测试 | 屏幕切换、座位渲染、角色可见性、交互面板 |
| 边界测试 | 极端场景（全死、自杀、首夜死人等） |
| 兼容性 | 浏览器直接打开运行 |

> 本项目为纯前端单文件游戏，无后端/API，不涉及安全/性能专项测试。

## 5. 已修复 Bug 记录

| Bug ID | 描述 | 修复内容 | 影响 |
|--------|------|---------|------|
| B-01 | `G.wolfSaveTarget` 变量名错误 | 改为 `G.witchSaveTarget` | 女巫解药逻辑完全失效 |
| B-02 | 死亡判定双重 if 检查冗余 | 合并为单层 `if(wolfTarget && witchSaveTarget !== wolfTarget)` | 逻辑正确但冗余，无实际影响 |
| B-03 | `revealed` 用于预言家内部查验 | 新增 `seerChecked` 区分查验与公开揭露 | 被查验玩家座位不该显示角色徽章 |
| B-04 | 狼人同伴在座位上不可见 | 添加条件 `(me.role===WOLF && me.alive && p.role===WOLF)` | 玩家狼人无法识别同伴 |
| B-05 | CSS `.seat.revealed` 规则未生效 | 渲染时添加 `seat.classList.add('revealed')` | 被揭露者座位角色徽章不显示 |

## 6. 测试执行方式

由于本项目是纯前端单文件游戏，所有测试均为手动执行：

1. 用浏览器打开 `011-cc-狼人杀/index.html`
2. 按 P0 用例顺序逐一执行
3. 每个用例需在不同角色下测试（可通过多次"再来一局"获得不同角色）
4. 边界测试需观察多局游戏结果

**建议测试顺序**: F-01~F-03 → F-04~F-08 → F-09~F-13 → F-14~F-22 → F-23~F-28 → F-29~F-33 → F-34~F-40 → F-41~F-48 → F-49~F-54 → U系列 → E系列
