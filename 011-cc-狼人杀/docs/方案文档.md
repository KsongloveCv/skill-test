# 狼人杀网页游戏 — 方案文档

## 1. 项目概述

| 项目 | 说明 |
|------|------|
| 名称 | 狼人杀（WEREWOLF · NIGHT OF THE DARK） |
| 类型 | 网页单机策略游戏（1名玩家 + 7名 AI） |
| 技术栈 | HTML + CSS + JavaScript，无框架依赖，单文件部署 |
| 入口文件 | `index.html`（浏览器直接打开即可运行） |
| 目录 | `011-cc-狼人杀/` |

## 2. 游戏配置

### 2.1 角色配置（8人标准局）

| 角色 | 数量 | 阵营 | 夜间技能 | 颜色标识 |
|------|------|------|----------|----------|
| 狼人 | 2 | 狼人阵营 | 选择一名玩家杀害 | 红色 `#e74c3c` |
| 预言家 | 1 | 好人阵营 | 查验一名玩家身份（狼/好人） | 蓝色 `#3498db` |
| 女巫 | 1 | 好人阵营 | 解药救人（1次）+ 毒药毒杀（1次） | 紫色 `#9b59b6` |
| 猎人 | 1 | 好人阵营 | 死亡时开枪带走一名玩家 | 橙色 `#e67e22` |
| 村民 | 3 | 好人阵营 | 无特殊技能，投票推理 | 绿色 `#2ecc71` |

### 2.2 玩家设定

- 0号位为玩家（"你"），其余7位为 AI
- 角色随机分配（Fisher-Yates 洗牌算法）
- AI 角色信息对玩家不可见，除非：
  - 玩家自身角色（始终可见）
  - 玩家是狼人时，同伴狼人可见
  - 玩家死亡后角色公开
  - 预言家公开跳身份指控

### 2.3 胜负判定

| 条件 | 结果 |
|------|------|
| 所有狼人死亡 | 好人阵营获胜 |
| 狼人数量 >= 好人数量 | 狼人阵营获胜 |

判定时机：每次有人死亡后立即检查。

## 3. 游戏流程（状态机）

```
开始 → 角色分配 → 角色揭示
  → [夜晚阶段]
    → 狼人杀人 → 预言家查验 → 女巫救/毒 → 结算死亡
    → 猎人开枪（如触发） → 胜负判定
  → [白天阶段]
    → AI 发言 → 玩家发言 → 投票放逐
    → 猎人开枪（如触发） → 胜负判定
  → 循环至胜负判定达成 → 结算屏幕
```

### 3.1 夜晚阶段

**狼人阶段（wolf）**
- 玩家是狼人：弹出选择面板，从非狼人存活玩家中选择目标
- AI 狼人：自动选择，优先杀预言家/女巫
- 两只狼人共同确定一个目标（当前简化为随机选择）

**预言家阶段（seer）**
- 玩家是预言家：弹出选择面板，查验目标后显示结果（狼/好人）
- AI 预言家：随机查验未查验过的玩家，标记 `seerChecked`

**女巫阶段（witch）**
- 玩家是女巫：
  - 第一步：看到被杀玩家，决定是否使用解药救人
  - 第二步：决定是否使用毒药毒杀某人
  - 解药和毒药各仅一次，使用后不可再用
- AI 女巫：
  - 第一夜/预言家被杀时大概率使用解药
  - 第二夜起 30% 概率使用毒药（50% 概率毒到狼人）

**死亡结算**
- 被狼人杀害 + 未被女巫解药救 → 死亡
- 被女巫毒药毒杀 → 亡
- 同一人被狼杀又被女巫毒 → 只死亡一次（不重复）
- 死亡角色公开揭示
- 猎人死亡触发开枪技能

### 3.2 白天阶段

**发言环节**
- 7名 AI 按角色策略生成发言文本
- 玩家选择预设发言选项（根据角色不同提供2-3个选项）

| 角色 | AI发言策略 |
|------|-----------|
| 狗人 | 伪装好人，指控无辜者 |
| 预言家 | 70%概率跳身份揭露狼人，30%保持沉默 |
| 女巫 | 提示药水状态，暗示关键信息 |
| 猎人 | 威慑狼人，表示死亡会开枪 |
| 村民 | 40%概率直觉指向狼人，否则随机怀疑 |

**投票放逐**
- 玩家从存活非自己玩家中选择投票目标，或弃票
- AI 投票策略：
  - 狼人：优先投已跳身份的预言家，否则投非狼人
  - 好人：优先投被揭露的狼人，否则随机投
- 最高票者被放逐（平票无人出局）
- 猎人被放逐时触发开枪技能

### 3.3 猎人技能触发

猎人无论在夜晚被杀还是白天被放逐，都可以开枪带走一名存活玩家：
- 玩家猎人：弹出选择面板
- AI 狼人：50%概率瞄准狼人，否则随机

## 4. 技术架构

### 4.1 状态模型

全局状态对象 `G` 包含：

```
G = {
  players: [{ id, role, name, avatar, alive, revealed, seerChecked, isMe, suspicion, votesReceived }],
  day: Number,          // 当前天数
  phase: 'night'|'day', // 当前阶段
  nightSub: 'wolf'|'seer'|'witch', // 夜间子阶段
  witchHasSave: Boolean,   // 女巫是否还有解药
  witchHasPoison: Boolean, // 女巫是否还有毒药
  wolfTarget: Number|null,     // 狼人目标
  seerTarget: Number|null,     // 预言家目标
  witchSaveTarget: Number|null, // 女巫解药目标
  witchPoisonTarget: Number|null, // 女巫毒药目标
  nightDeaths: Number[],  // 当夜死亡列表
  messages: [{speaker, text, cls}], // 消息日志
  speeches: [{playerId, text}],     // 发言记录
  gameOver: Boolean,
  winner: 'wolf'|'good'|null,
  selectedId: Number|null, // 当前选中目标
}
```

### 4.2 属性说明

| 属性 | 含义 |
|------|------|
| `alive` | 玩家是否存活 |
| `revealed` | 身份是否被公开揭露（死亡、预言家跳身份） |
| `seerChecked` | 预言家是否已查验（仅预言家内部可见，不影响UI显示） |
| `isMe` | 是否为玩家角色（0号位） |
| `suspicion` | 可疑度（预留，当前未用于投票权重） |
| `votesReceived` | 当轮投票得票数（预留） |

### 4.3 核心函数清单

| 函数 | 用途 |
|------|------|
| `initState()` | 初始化游戏状态，洗牌分配角色 |

## 5. UI 设计

### 5.1 视觉风格

- **色调**: 暗色哥特风格，主色 `#0a0a0f`，强调色红色 `#e74c3c`
- **背景**: Canvas 粒子动画（夜晚蓝色粒子，白天暖色粒子）+ 渐变天空覆盖
- **字体**: 'Segoe UI' / system-ui / sans-serif
- **按钮**: 圆角胶囊型（border-radius: 50px），hover 时发光上移

### 5.2 座位布局

- 8个座位按环形排列（520x520px 容器，半径200px）
- 每个座位 100x120px，包含：编号、头像emoji、姓名、角色徽章
- 座位状态样式：
  - `is-me`: 金色边框+发光（玩家自己）
  - `dead`: 灰色半透明+骷髅标记+头像灰度化
  - `selected`: 红色边框+发光（选中目标）
  - `{role}-seat`: 对应角色颜色边框

### 5.3 角色徽章可见性规则

```
visible = p.isMe            // 玩家自己始终可见
       || !p.alive          // 死亡后公开
       || p.revealed        // 被预言家跳身份公开揭露
       || (me.role===WOLF && me.alive && p.role===WOLF)  // 狼人同伴互认
```

不可见时显示 `hidden-role`（灰色，文字"???").

### 5.4 动画效果

| 效果 | 实现方式 |
|------|----------|
| 角色卡翻转 | CSS `@keyframes cardFlip` — rotateY 90°→0° + scale |
| 屏幕切换 | opacity + transform:scale 过渡 0.6s |
| 天空渐变 | CSS transition 2s 背景颜色切换 |
| 粒子动画 | Canvas requestAnimationFrame 循环 |
| 座位hover | CSS transform:scale(1.05) 0.3s |
| 按钮hover | translateY(-2px) + box-shadow 发光 |
| 夜晚/白天 | 中央面板 + 阶段条 + 天空三层联动 |
| Toast弹出 | opacity 0→1 + 2s自动消失 |

## 6. 玩家交互设计

### 6.1 夜间操作（按角色）

| 玩家角色 | 操作 | UI |
|---------|------|-----|
| 狼人 | 从非狼存活玩家中选择杀人目标 | `night-overlay` 选择面板 |
| 预言家 | 从存活非自己玩家中选择查验目标 | `night-overlay` 选择面板 + Toast显示结果 |
| 女巫 | 先选解药(救人/不救)，再选毒药(毒人/不毒) | 两阶段 `night-overlay` |
| 猎人 | 无夜间操作 | 自动跳过 |
| 村民 | 无夜间操作 | 自动跳过 |

### 6.2 白天操作

| 操作 | UI |
|------|-----|
| 发言 | `action-panel` 提供角色特定发言选项 |
| 投票 | `vote-overlay` 卡片网格选择 + 确认/弃票按钮 |
| 猎人开枪 | `night-overlay` 选择带走目标 |

### 6.3 非玩家阶段

玩家不是当前操作角色时，AI 自动执行，界面显示简短系统消息（如"狼人已选择目标"），1.5秒后进入下一阶段。

## 7. AI 决策策略

### 7.1 狼人

```
优先目标: 预言家 > 女巫 > 其他好人
如果预言家/女巫存活 → 从其中随机选一个
否则 → 从所有非狼人存活玩家中随机选
```

### 7.2 预言家

```
查验优先: 未被查验过的玩家(seerChecked===false) > 随机
发言概率: 70%跳身份揭露狼人, 30%保持沉默
```

### 7.3 女巫

```
解药: 第一夜或预言家/猎人被杀时使用
毒药: 第二夜起30%概率使用
  - 若使用, 50%概率瞄准狼人(运气成分), 50%随机
  - 不会毒自己, 不会毒已救目标
```

### 7.4 猎人

```
开枪: 50%概率瞄准狼人, 50%随机
威慑发言: "如果我死了，一定会带走一个人"
```

### 7.5 村民

```
发言: 40%概率直觉指向狼人(模拟推理), 60%随机怀疑
投票: 投被揭露的狼人 > 随机投
```

### 7.6 投票共性

- 狼人：优先投已跳身份预言家，否则投非狼人
- 好人：优先投被揭露(`revealed`)的狼人，否则随机投
- 所有AI不会投自己

## 8. 消息系统

- 所有事件写入右侧 `msg-panel` 日志面板
- 消息按角色着色：狼人红色、预言家蓝色、女巫紫色、猎人橙色
- 系统消息无颜色前缀
- 重大事件同步弹出 Toast 提示（顶部2秒）
