# 文章爬取与润色 Skill

## 功能概述
本 Skill 能够：
1. 从用户提供的网页 URL 中提取文章标题和正文内容
2. 对提取的正文进行智能润色（优化语句、修正语法、提升可读性）
3. 基于内容自动生成关键词和标签
4. 以结构化格式输出完整结果，并附上原文链接供参考

## 使用场景
- 需要快速获取并优化网络文章内容
- 为内容创作提供素材和灵感
- SEO 关键词提取与标签生成
- 内容二次加工与改写

## 输入参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| url | string | 是 | 目标文章的完整 URL |
| language | string | 否 | 文章语言（默认自动检测） |
| max_keywords | integer | 否 | 关键词数量（默认 5） |
| max_tags | integer | 否 | 标签数量（默认 3） |

## 输出格式
【标题】
{润色后的标题}

【正文】
{润色后的正文内容}

【关键词】
关键词1,关键词2,关键词3,关键词4,关键词5

【标签】
标签1,标签2,标签3

【原文链接】
{原始文章URL}


## 工作流程

### Step 1: 网页内容抓取
- 使用 `requests` 库发送 HTTP GET 请求获取网页 HTML
- 设置合理的 User-Agent 模拟浏览器访问
- 处理常见反爬机制（添加延迟、重试机制）
- 支持 UTF-8 编码自动识别

### Step 2: 内容提取
- 使用 `BeautifulSoup` 解析 HTML
- 智能识别文章主体内容区域：
  - 优先识别 `<article>`、`<main>` 标签
  - 备选方案：基于常见 class/id 名称（如 `.post-content`、`#content`、`.entry-content`）
  - 使用 readability 算法提取核心正文
- 提取标题（优先 `<h1>` 或 `og:title` meta 标签）
- 清除无关元素（广告、导航、侧边栏、脚本、样式）

### Step 3: 文本清洗
- 移除多余空白字符和换行符
- 保留段落结构（使用双换行分隔段落）
- 过滤短小无意义的片段
- 处理特殊字符和 HTML 实体

### Step 4: 内容润色
调用大语言模型进行内容优化，遵循以下原则：
- 保持原意不变，仅改善表达
- 修正语法错误和拼写错误
- 优化句子结构，提升流畅度
- 适当调整段落划分，增强可读性
- 标题可进行轻度优化（更吸引人但不失准确性）

**润色 Prompt 模板：**
请对以下文章内容进行专业润色，要求：
    保持原意和关键信息完整
    修正语法、拼写、标点错误
    优化句式结构，提升流畅度和专业性
    段落划分合理，层次清晰
    不要添加原文没有的新内容
    输出纯文本，保持段落用两个换行分隔

原文标题：{title}
原文内容：
{content}

请返回润色后的标题和正文，格式如下：
【标题】
润色后的标题
【正文】
润色后的正文


### Step 5: 关键词和标签提取
基于润色后的内容，自动生成 SEO 友好的关键词和标签：

- **关键词**：反映文章核心主题的短语（2-4 个词）
- **标签**：更宽泛的分类词或热门话题词（1-2 个词）

**提取 Prompt 模板：**
请根据以下文章内容，提取：
    {max_keywords} 个关键词（2-4 个词的短语，反映核心主题）
    {max_tags} 个标签（1-2 个词的分类词或热门话题）

要求：
    关键词和标签都使用英文逗号分隔
    关键词和标签不要重复
    只输出最终的逗号分隔列表，不要其他解释

文章内容：
{rewritten_content}

输出格式示例：
关键词1,关键词2,关键词3
标签1,标签2

### 使用方法
在 Claude 对话中直接调用本 Skill，并提供目标 URL：
请使用 article-scraper-rewriter skill 处理这个链接：https://example.com/article


### 异常处理
- 网络请求失败：返回明确错误信息，建议用户检查 URL
- 内容提取失败：尝试备用提取方法，若仍失败则提示手动提供内容
- LLM API 调用失败：降级方案为仅清洗不润色，或返回原始内容

## 注意事项
1. 遵守目标网站的 robots.txt 规定
2. 设置合理的请求间隔，避免对服务器造成压力
3. 尊重版权，仅用于个人学习和合理引用
4. 大语言模型润色结果需人工复核，确保信息准确性
